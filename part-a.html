01 — Dataset

The TMDB Dataset

Data is sourced from the TMDB (The Movie Database) API. The discovery endpoint returns popular films sorted by popularity; we then fetch full details and cast credits for each. Filtering criteria ensure the network is large enough to study but feasible to collect.

Collection Parameters

  • Release window2010 – 2025
  • Min. vote count≥ 200
  • Discover pages10 pages
  • Sort orderpopularity.desc
  • Cast rank cutofftop-8 per film

Why These Filters?

  • Vote count filter removes niche films with sparse cast data
  • Cast rank keeps the analysis centered on visible actors
  • Recent window (2010–2025) ensures relevant production landscape

02 — Pipeline

Collection & Processing Pipeline

The pipeline is structured around six sequential stages. JSON caching ensures that API calls are not re-executed on notebook reruns, keeping collection reproducible and cost-efficient.

01

Discover Movies

Query /discover/movie with filters. Paginate through results to collect movie stubs.

02

Fetch Details

For each movie, call /movie/{`{id}`} to get genres, overview, vote stats.

03

Get Cast Credits

Call /movie/{`{id}`}/credits and keep actors with cast_order ≤ 7.

04

Build DataFrames

Combine raw JSON into three tidy tables: movies, actors, and actor-movie links.

05

Construct Graph

Enumerate co-cast pairs per movie; edge weight counts shared co-appearances.

06

Analyze & Export

Run network stats, community detection, and text analysis. Save all artifacts.

All API responses are cached as JSON files in data/raw/ using a hash-based naming scheme. A 0.2-second sleep between calls respects TMDB rate limits, and exponential backoff handles 429 responses automatically.

03 — Network

Actor Co-appearance Network

The network is an undirected, weighted graph where nodes are actors and edges represent shared film appearances. This captures collaboration intensity — not just whether two actors have worked together, but how often.

Network Definition

Nodes

One per unique actor appearing in the top-8 cast of any collected film. Node attributes include name, gender, department, and popularity.

Edges

An edge (u, v) exists when actors u and v both appear in at least one shared film. Edge weight = number of co-appeared films.

Construction

For each film, enumerate all C(k,2) actor pairs where k ≤ 8. Increment edge weight by 1 per co-occurrence using a Counter.

Network Statistics

Density ~0.007

Degree distribution Right-skewed

Low density (~0.7%) is typical for collaboration networks — a giant connected component contains the majority of nodes, with sparse edges reflecting real-world collaboration patterns.

04 — Text

Movie Overview Text Analysis

Each TMDB movie record includes a short plot summary (overview). These summaries are preprocessed and analyzed to understand the thematic language associated with different actor communities.

Text Preprocessing Steps

  1. 1 Lowercase conversion
  2. 2 Remove URLs and non-alphabetic characters
  3. 3 Filter tokens shorter than 3 characters
  4. 4 Remove English stopwords (the, and, for…)
  5. 5 Domain-specific stopword removal (film, story, find…)

Analysis Methods

Term Frequency (TF)

Raw count of each token per community document. Identifies the most common words in each cluster's movie plots.

TF-IDF

Weights each term by its uniqueness across communities. Surfaces distinctive vocabulary that characterizes each community versus the rest.

05 — Integration

Connecting Network to Text

The key innovation is linking Louvain community labels back to movie overview text — allowing us to compare what kinds of stories different actor clusters tend to appear in.

Louvain Community Detection

Run on the Giant Connected Component (GCC) of the actor network using edge weights as modularity input. Returns a partition mapping each actor to a community ID.

Actor → Community Mapping

Each actor_id in the actor_movie_df table receives its community label via a join. This extends community membership to every film appearance.

Community Document Aggregation

All movie overview texts for films linked to a given community are concatenated into a single corpus document — one document per community.

TF and TF-IDF Analysis

Term frequencies and TF-IDF scores are computed per community document. Top-scoring terms reveal the distinctive themes and vocabulary of each actor cluster.

06 — Findings

Preliminary Findings

Scalable Pipeline

All three DataFrames were validated with zero null keys and zero duplicate rows after collection and cleaning.

Dense GCC Formed

The Giant Connected Component encompasses the large majority of nodes, confirming the network is well-connected enough for community analysis.

Thematic Variation Observed

Initial TF analysis shows communities using distinct sets of vocabulary in their plot summaries — consistent with genre specialization.

Limitations Identified

Popularity bias in TMDB discovery over-represents mainstream productions. Cast rank cutoff may exclude important secondary actors from some communities.
